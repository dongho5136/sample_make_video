# 광고 생성: 구간별 카테고리 선택 — 설계 문서

작성일: 2026-07-05

## 1. 목적

광고 생성 시 템플릿을 고르면 **구간(segment)마다** 그 템플릿의 **모든
카테고리**에서 요소를 각각 골라 광고를 구성할 수 있게 한다. 구간별로 캐릭터·
문구·장면 등을 다르게 선택해 "구간별로 만드는" 흐름을 표현한다.

현재는 템플릿당 카테고리를 한 세트만 고르고 전체가 순차 생성되는 애니메이션만
있다. 이를 **구간 × 카테고리** 선택 구조로 확장한다.

## 2. 결정 사항 (브레인스토밍 확정)

- 구간에서 선택하는 요소: **템플릿의 모든 카테고리**를 구간마다 각각
- 구간 수: **길이 자동** — 15초→2, 30초→4, 60초→6 (기존 규칙 유지)
- 구간 이름: 고정 시나리오(후킹/문제상황/제품소개/사용장면/혜택강조/CTA)에서
  앞에서부터 슬라이스 (기존 `SEGMENT_SCENARIO` 재사용)
- UI 배치: **구간별 카드/섹션 세로 나열**
- 카테고리 선택: **선택사항** (비워도 생성 가능)

## 3. 화면 구성

```
[스타일 템플릿 ▾]                          🎲 전체 랜덤

┌─ 1. 후킹                            🎲 ─┐
│  캐릭터 [▾]   후킹 문구 [▾]   장면 구성 [▾]│
├─ 2. 문제상황                        🎲 ─┤
│  캐릭터 [▾]   후킹 문구 [▾]   장면 구성 [▾]│
├─ 3. 제품소개                        🎲 ─┤
│  ...                                     │
└──────────────────────────────────────────┘

[🎬 생성 요청]  →  우측 진행 상태 애니메이션(기존)
```

- 템플릿 미선택: "템플릿을 먼저 선택하세요" 안내
- 템플릿에 카테고리 없음: 각 구간 카드에 "선택할 요소가 없습니다" 안내
- 각 구간 카드: `번호. 구간이름` 헤더 + 구간별 🎲 + 카테고리 드롭다운들
- 상단 "전체 랜덤": 모든 구간의 모든 카테고리를 한 번에 랜덤

## 4. 동작

- **구간 수 계산**: `SEG_COUNT = {15:2, 30:4, 60:6}`, 없으면 4. `SEGMENT_SCENARIO`
  앞에서부터 슬라이스해 구간 이름 배열 생성.
- **길이/템플릿 변경 시**: 구간 카드 재생성 (선택값 초기화).
- **구간별 🎲**: 그 구간의 모든 카테고리 드롭다운을 각각 무작위 선택.
- **전체 랜덤**: 모든 구간 × 모든 카테고리 무작위 선택.
- **생성 요청**:
  - 필수 검증(기존 유지): 광고명·제품명·템플릿. 카테고리 선택은 선택사항.
  - payload: `{name, product, brand, purpose, length, ratio, model, template,
    note, segments: [{name, selections: {카테고리명: 항목명, ...}}, ...]}`
  - `POST /api/jobs` (기존, 더미 성공 응답)
  - 진행 애니메이션: 구간 배열을 순차적으로 `대기중→생성중→완료` 전환 후
    `합본 생성 중→완료` (기존 로직 유지, 구간 이름을 segments에서 사용)

## 5. 영향 범위

- 수정:
  - `templates/generate.html` — 단일 "카테고리 선택" 영역 → "구간 컨테이너"로 교체
  - `static/js/generate.js` — 구간별 렌더/랜덤/수집 로직으로 재작성
- 변경 없음: `data/dummy.py`, `app.py`(라우트/API), `scripts/smoke.py`
- 데이터/API: 기존 `GET /api/templates`(categories), `GET /api/meta`(scenario),
  `POST /api/jobs` 재사용

## 6. 에러 처리

- 템플릿 미선택 시 생성 버튼 누르면 템플릿 필드 인라인 에러(기존 유지)
- 카테고리 없는 템플릿: 구간 카드에 안내만, 생성은 정상 진행(선택 없이)
- fetch 실패: 기존 패턴대로 처리(초기 로드 실패 시 콘솔 오류 없이 빈 셀렉트)

## 7. 검증

- `scripts/smoke.py`: 라우트 변경 없으므로 기존 스모크 그대로 통과 확인
- Playwright 육안:
  - 템플릿 선택 → 구간 카드 N개 생성
  - 광고 길이 15/30/60 변경 → 구간 수 2/4/6 반영
  - 구간별 🎲, 전체 랜덤 동작 → 드롭다운 값 채워짐
  - 생성 요청 → 진행 애니메이션(구간 순차 완료 + 합본 완료 토스트)
- 프로덕션 curl 스모크 후 Vercel 재배포

## 8. 배포

구현·검증 완료 후 GitHub push → `vercel deploy --prod`.
