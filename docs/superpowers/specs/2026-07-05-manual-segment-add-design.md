# 광고 생성: 구간 수동 추가/삭제 — 설계 문서

작성일: 2026-07-05

## 1. 목적

지금은 광고 길이(15/30/60)로 구간 수가 자동 결정된다. 이를 **"+ 구간 추가"
버튼으로 구간을 하나씩 직접 추가**하고, 각 구간에서 요소를 고르는 방식으로
바꾼다. 구간은 개별 삭제도 가능하다.

## 2. 결정 사항 (브레인스토밍 확정)

- 구간 이름: **시나리오 순서 자동** — 후킹→문제상황→제품소개→사용장면→혜택강조
  →CTA, 6개 초과 시 "구간7", "구간8"…
- 광고 길이–구간 관계: **분리** — 길이는 메타 정보로만, 구간 수와 무관
- 구간 삭제: **구간별 ✕ 삭제 버튼** (삭제 시 번호 자동 재정렬)
- 템플릿 최초 선택 시: **1구간 자동 생성**

## 3. 화면 구성

```
[스타일 템플릿 ▾]            🎲 전체 랜덤   [+ 구간 추가]

┌─ 1. 후킹              🎲  ✕ ─┐
│  캐릭터[▾] 후킹문구[▾] 장면[▾] │
├─ 2. 문제상황          🎲  ✕ ─┤
│  캐릭터[▾] 후킹문구[▾] 장면[▾] │
└──────────────────────────────┘

[🎬 생성 요청]  → 우측 진행 상태 애니메이션(기존)
```

- 템플릿 미선택: "템플릿을 먼저 선택하세요"
- 템플릿 선택 시: 구간 1개 자동 생성
- 각 구간 카드: `번호. 이름` 헤더 + 구간별 🎲 + ✕ 삭제 + 카테고리 드롭다운들
- 카테고리 없는 템플릿: 구간 카드에 "선택할 요소가 없습니다"

## 4. 동작

- **구간 상태 관리**: JS에 `segmentList = [{name}]` 배열 유지. 렌더는 이 배열 기준.
- **이름 규칙**: `scenarioName(index)` — `SEGMENT_SCENARIO[index]` 있으면 그 값,
  없으면 `"구간" + (index+1)`.
- **템플릿 change**: `segmentList = [{name: scenarioName(0)}]` (1구간 자동) 후 렌더.
  단, 드롭다운 선택값은 렌더 시 초기화(템플릿 바뀌면 카테고리도 달라짐).
- **"+ 구간 추가"**: `segmentList.push({name: scenarioName(segmentList.length)})` 후 렌더.
- **✕ 삭제**: 해당 인덱스 제거 후 이름 재계산(scenarioName 재적용)하여 재렌더.
  구간 번호(1., 2., …)는 렌더 시 인덱스로 매김.
- **광고 길이**: 메타 정보로만 payload에 포함. 구간 수와 무관(length change로
  구간 재생성하지 않음).
- **전체 랜덤 / 구간 랜덤 / 수집 / 진행 애니메이션**: 기존 로직을 `segmentList`
  기준으로 동작하도록 유지.
- **생성 요청**: 필수 검증(광고명·제품명·템플릿) 유지. 구간이 0개여도 생성 가능
  (합본만). payload `segments: [{name, selections}]`.

## 5. 영향 범위

- 수정:
  - `templates/generate.html` — "구간별 요소 선택" 헤더에 "+ 구간 추가" 버튼 추가
  - `static/js/generate.js` — `segmentList` 상태 + 추가/삭제/이름 로직으로 재작성,
    길이 기반 자동 구간 생성 제거, 삭제 버튼 처리
- 변경 없음: `data/dummy.py`, `app.py`, `scripts/smoke.py`
- 데이터/API: 기존 `GET /api/templates`, `GET /api/meta`, `POST /api/jobs` 재사용

## 6. 에러 처리

- 템플릿 미선택 후 생성: 템플릿 필드 인라인 에러(기존)
- 카테고리 없는 템플릿: 구간 카드 안내만, 생성 정상 진행
- 구간 삭제로 0개가 되어도 생성 가능(합본 애니메이션만)

## 7. 검증

- `scripts/smoke.py`: 라우트 변경 없음 → 기존 스모크 그대로 통과
- Playwright 육안:
  - 템플릿 선택 → 1구간 자동 생성
  - "+ 구간 추가" 반복 → 2,3,4… 이름/번호 자동 부여
  - ✕ 삭제 → 번호/이름 재정렬
  - 구간별 🎲 / 전체 랜덤 → 값 채워짐
  - 생성 요청 → 진행 애니메이션(구간 순차 + 합본 완료)
- 프로덕션 curl 스모크 후 Vercel 재배포

## 8. 배포

구현·검증 완료 후 GitHub push → `vercel deploy --prod`.
